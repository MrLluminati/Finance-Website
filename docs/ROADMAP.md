# Roadmap

## Roadmap Philosophy

This project should grow in layers. Each layer should be useful on its own, portfolio-worthy, and safe to build within a low or zero budget.

The roadmap avoids starting with real-time stock data because that creates licensing, cost, and compliance problems too early.

---

# Phase 0: Planning and Repository Setup

## Objective

Create a clear foundation before coding.

## Deliverables

- README
- Project plan
- Roadmap
- Compliance/data-use policy
- GitHub issues for first tasks
- Basic product scope

## Status

In progress.

# Phase 1: Static Finance Portal MVP

## Objective

Create a polished public finance website with strong UI, SEO structure, and useful educational content.

## Features

- Homepage
- Markets-style overview layout
- News/Insights section
- Learn section
- Tools index
- Stocks/company directory
- Basic company profile template
- Disclaimer, Privacy Policy, Terms
- Responsive mobile-first design

## Technical Work

- Initialize Next.js app
- Add Tailwind CSS
- Create layout components
- Create reusable cards and sections
- Add static content through MDX or structured JSON
- Deploy to Vercel or similar free hosting

## Success Criteria

- Public deployed website
- Mobile responsive
- Clean GitHub repository
- At least 3 working finance calculators
- At least 5 company profile pages
- At least 5 educational articles/explainers

# Phase 3: SaaS Foundation

## Objective

Introduce user-specific functionality.

## Features

- User authentication
- Dashboard
- Saved watchlist
- Saved articles
- User notes
- Manual portfolio tracker MVP

## Technical Work

- Supabase or equivalent backend
- User tables
- Watchlist tables
- Portfolio tables
- Protected routes
- Dashboard UI

## Success Criteria

- Users can create an account
- Users can save companies
- Users can manually create a portfolio
- User data persists in database

# Phase 4: Portfolio Analytics

## Objective

Make the dashboard genuinely valuable.

## Features

- Manual transaction entry
- Average price calculation
- Allocation charts
- Sector exposure
- Asset-class distribution
- Notes per holding
- Profit/loss calculator using manually entered or delayed/free data where legally permitted

## Technical Work

- Portfolio data model
- Chart components
- Aggregation logic
- Export-friendly data structure

## Success Criteria

- User can track portfolio manually
- User can see allocation and basic analytics
- System remains legally safe and transparent about data limitations

# Phase 6: Monetization Layer

## Objective

Make the platform self-sustaining.

## Possible Revenue Features

- Ads
- Newsletter sponsorships
- Premium dashboard
- Advanced tools
- PDF company reports
- Premium research notebook
- Affiliate partnerships where legally safe

## Caution

Do not monetize investment recommendations unless regulatory compliance has been reviewed.

---

# Phase 7: Licensed Market Data and Advanced Platform

## Objective

Move toward a serious finance data platform.

## Features

- Licensed data feed integration
- Real-time or delayed market data, depending on license
- Advanced watchlists
- Alerts
- Screeners
- API layer
- Mobile app possibility

## Requirement

This phase requires proper data licensing, infrastructure budget, and legal review.

---

# Long-Term Vision

The final platform may include:

- global market coverage;
- multilingual finance education;
- portfolio analytics;
- financial planning tools;
- regulation-aware financial content;
- AI-assisted explainers;
- premium research workspace;
- licensed market data;
- trusted editorial standards.

The project must grow from credibility, not shortcuts.
